Implementation Guide

1

Add the debugger script to your page

Include the enhanced flag debugger script in your HTML file:

<script src="enhanced-flag-debugger.js"></script>

Add this script after your PostHog initialization code to ensure it can detect PostHog properly.

2

Enable the debugger with URL parameters

The debugger only activates when you include ?debugPHFlags in your URL. This ensures regular users don't see debugging information.

https://yourdomain.com/yourpage?debugPHFlags

By default, this will only log to the console and check for the sandbox_embed flag with an expected value of with-sandbox.

3

Configure the debugger using URL parameters

Customize the debugger behavior with additional parameters:

https://yourdomain.com/yourpage?debugPHFlags&flag_name=my_flag&flag_value=true&selector=.my-element&mode=window

Available parameters:

  • flag_name: The name of the feature flag to check (default: sandbox_embed)
  • flag_value: The expected value of the flag (default: with-sandbox)
  • selector: CSS selector for the element to check/inject (default: .homepage-sandbox)
  • container: Where to inject a mock element if needed (default: body)
  • mode: Set to window to display the visual debugger (default: console)
4

Analyze the results

The debugger provides detailed information in two ways:

  1. Console logs: Check your browser's developer tools (F12) for timing data and status updates.
  2. Visual debugger: If mode=window is set, a debugger panel appears in the bottom-right corner with real-time status indicators.

You can also access the debug data programmatically:

// Get complete debug report
window.PostHogDebugger.getDebugReport()

// Toggle visibility of the debugger panel
window.PostHogDebugger.toggleVisibility()

Common Use Cases

Troubleshoot Race Conditions

Identify timing issues between PostHog flag loading and DOM element visibility.

URL configuration:

?debugPHFlags&mode=window

Test Specific Flags

Check if a particular feature flag is loading with the expected value.

URL configuration:

?debugPHFlags&flag_name=custom_feature&flag_value=enabled

Validate Element Visibility

Confirm that specific elements are properly shown/hidden based on flags.

URL configuration:

?debugPHFlags&selector=.feature-container&mode=window
